# M1: Local Governance Baseline

## Scope

Build the local governance baseline for the agent workflow system.

M1 defines the canonical task contracts, evidence layout, local Bridge commands, and closeout flow needed to decide whether work is accepted, failed, or inconclusive.

## Non-Goals

Do not implement:

- Multica integration
- GitHub Issue integration
- GitLab integration
- dashboard
- auto-merge
- PR creation automation
- model API calls
- autonomous long-running loops
- worker metrics routing
- bidirectional sync
- production secret handling
- VPN/internal network automation

## Evidence Pointer

Structured evidence lives under:

```text
docs/milestones/evidence/<task_id>/
```

## Closeout Pointer

Closeout lives under:

```text
docs/milestones/closeout/
```

## Risks

- Bridge is intentionally local-only in M1.
- Evidence quality depends on structured files being written correctly.
- Reviewer and verifier roles are process roles; Bridge does not call models or agents.

## Evidence summary

## Accepted tasks

## Failed or inconclusive tasks
