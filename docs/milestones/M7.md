# M7: Multica Live Execution Evidence POC

## Scope

Prove that a real or manually represented Multica task can become local Coshare evidence while Bridge gate remains the final acceptance authority.

M7 uses manual export or offline fixture evidence only.

Bridge does not call Multica APIs.

## Non-Goals

Do not implement:

- full Multica integration
- Multica API calls from Bridge
- creating or updating Multica tasks from Bridge
- reading Multica live state inside Bridge gate
- writing Multica comments from Bridge
- treating Multica task state as canonical
- treating Multica completed as accepted
- storing Multica tokens
- scheduler
- worker registry
- dashboard
- autonomous loop
- bidirectional sync
- automatic PR creation
- push automation
- auto-merge
- model API calls
- GitLab integration
- GitHub Projects integration
