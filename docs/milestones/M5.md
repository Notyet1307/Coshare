# M5: GitHub Delivery Linkage and Acceptance Publication

## Scope

Build GitHub Delivery Linkage and Acceptance Publication for the agent workflow governance system.

M5 keeps milestone docs as the canonical task source.

GitHub Issues, PRs, CI checks, reviews, comments, and labels are evidence or publication surfaces only.

Bridge gate remains the final acceptance authority.

## Non-Goals

Do not implement:

- GitHub Issue as canonical task source
- automatic milestone-doc updates from GitHub Issue body
- GitHub Projects integration
- GitLab integration
- Multica integration
- dashboard
- auto-merge
- automatic PR creation
- push automation
- model API calls inside Bridge
- autonomous long-running loops
- worker metrics routing
- bidirectional sync
- production secret handling
- storing GitHub tokens in repo files
- VPN/internal network automation

## Evidence Pointer

Structured evidence lives under:

```text
docs/milestones/evidence/<task_id>/
```

## Closeout Pointer

Closeout lives under:

```text
docs/milestones/closeout/
```
