# M4: GitHub Issue Execution Ticket Adapter

## Scope

Build the GitHub Issue Execution Ticket Adapter for the agent workflow governance system.

M4 keeps milestone docs as the canonical task source.

GitHub Issues are execution tickets and mirrors only.

## Non-Goals

Do not implement:

- GitHub Issue as canonical task source
- issue-to-doc sync
- automatic issue creation by default
- live GitHub writes without explicit user request
- GitHub Projects integration
- GitLab integration
- Multica integration
- dashboard
- auto-merge
- automatic PR creation
- push automation
- model API calls inside Bridge
- autonomous long-running loops
- worker metrics routing
- bidirectional sync
- production secret handling
- storing GitHub tokens in repo files
- VPN/internal network automation

## Evidence Pointer

Structured evidence lives under:

```text
docs/milestones/evidence/<task_id>/
```

## Closeout Pointer

Closeout lives under:

```text
docs/milestones/closeout/
```
