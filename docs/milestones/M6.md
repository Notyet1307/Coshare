# M6: Execution Platform Delegation POC

## Scope

Prove that Coshare can consume external execution-platform state as local evidence while keeping Bridge gate as the final acceptance authority.

M6 is an offline planning and boundary POC only.

External completed does not equal accepted.

## Non-Goals

Do not implement:

- live Multica integration
- Multica API calls
- GitLab integration
- dashboard
- scheduler
- worker registry
- autonomous loop
- automatic PR creation
- push automation
- auto-merge
- live GitHub writes
- model API calls inside Bridge
- token or secret storage
- VPN/internal network automation

## Evidence Pointer

## Phase 7 Decision Options

- Multica live POC: use if the offline boundary works and Multica is the likely execution substrate.
- OpenHands POC: use if local runtime ownership matters more than issue/task queue UX.
- GitHub-only: use if GitHub Issues and PRs provide enough coordination.
- Coshare-only: use only if external execution coordination adds more complexity than value.
